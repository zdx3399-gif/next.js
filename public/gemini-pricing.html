一推薦採用的付費方案與精確模型

切換至 Pay-as-you-go（按量計費） 方案後，可解除每分鐘 15 次請求的 Free Tier 限制，適用商業 SaaS 部署。 內容審查任務推薦以下兩款最新 Flash 模型：

gemini-2.5-flash-lite ⭐ 核心推薦

適用場景：常規論壇貼文、留言過濾，需應對瞬間高併發。

優勢：極致性價比，延遲最低，吞吐量最大。

輸入 (Input) $0.10 USD / 1M tokens
輸出 (Output) $0.40 USD / 1M tokens
脈絡快取 (Context Cache) $0.05 USD / 1M tokens
gemini-2.5-flash 進階備用

適用場景：陰陽怪氣、高級酸、暗喻辱罵等 Lite 模型難以判定的情境。

優勢：推理能力更強，語境理解更細膩。

輸入 (Input) $0.30 USD / 1M tokens
輸出 (Output) $2.50 USD / 1M tokens
脈絡快取 (Context Cache) —

三核心計費公式拆解（供查證）
總花費公式
\[ \text{每月總花費} = \bigl(\text{請求數} \times \text{輸入 tokens} \times \text{輸入單價}\bigr) + \bigl(\text{請求數} \times \text{輸出 tokens} \times \text{輸出單價}\bigr) \]

以 gemini-2.5-flash-lite 處理 30,000 次請求為例：

輸入成本 → 30,000 × 500 tokens = 15,000,000 tokens → 15M × $0.10 = $1.50
📌 若命中脈絡快取（Context Caching），費率降至 $0.05 / 1M tokens，成本可減至 $0.75
輸出成本 → 30,000 × 50 tokens = 1,500,000 tokens → 1.5M × $0.40 = $0.60
調整後實際總計 → 約 $0.45 ~ $2.10 USD （依快取命中率而定）
四結論與開發步驟
1
初期開發與測試 — 在 Google AI Studio 使用 Free Tier（免費層） 進行 Prompt 調整， 模型選 gemini-2.5-flash-lite，零成本驗證審查邏輯。
2
正式上線營運 — 將 Google Cloud 專案綁定信用卡，升級為 Pay-as-you-go 方案， 徹底解鎖每分鐘 15 次限制，持續使用 gemini-2.5-flash-lite。
3
模型效能評估 — 上線一週後檢查管委後台「誤判率」。 若陰陽怪氣、暗喻辱罵無法精準判定，只需將後端的 Model Name 改為 gemini-2.5-flash 即可無縫升級，無需任何架構調整。
